# Moongate 0.8.1 Subtitle Human Quality ExecPlan

## Background And Product Intent

Moongate 0.8.1 already has local-ASR timing, subtitle source selection, scorecard, and viewing-quality tooling. The next product target is not only "Whisper text is accurate"; it is whether the final subtitle experience feels close to a human-authored subtitle across mainstream languages, audio-only samples, source selection, and translation output.

The long-term target is 90%+ human-level subtitle quality, with an ideal target of 95%+. The first implementation milestone is a stronger eval baseline that can expose first-word onset timing, source decision quality, segmentation health, recognition quality, and translation verification without changing production subtitle algorithms yet.

## Current Repository Understanding

- `tools/subtitle_timing_eval/subtitle_timing_eval/scorecard.py` already scores recognition, segmentation, translation, and source decision.
- `tools/subtitle_timing_eval/run_scorecard_baseline.py` scans ignored artifacts and writes `scorecard.json` / `scorecard.md`.
- `run_scorecard_baseline.py` now scores the selected final source from `source_candidates.json` when that evidence exists, and falls back to `local-asr.*.srt` only when no source candidate report is present.
- `tools/subtitle_timing_eval/subtitle_timing_eval/segmentation.py` measures reference segmentation similarity, but reference-style segmentation is informational for scorecard because prior evals showed YouTube-style line breaks cap Whisper-style segmentation.
- `Sources/moongate-cli/main.swift` provides the local `local-asr-srt` regeneration path used by eval scripts. It now accepts both Moongate words JSON and raw whisper.cpp JSON, which lets eval regenerate subtitles through the same parser path production uses.
- `Sources/MoongateCore/SubtitleSourceDecisionEngine.swift` and `windows/MoongateCore/SubtitleSourceDecisionEngine.cs` are the production authority for platform/manual/local-ASR source decisions.
- Media and generated subtitles belong under ignored `artifacts/subtitle_timing_eval/`; do not commit downloaded audio/video or large subtitle corpora.

## Goals

1. Keep all work on `feature/subtitle-human-quality-0.8.1`.
2. Add a first-word onset metric to the scorecard so the first cue can be judged against the first available ASR word or speech segment.
3. Preserve the existing scorecard contract: unverified structural scores can guide debugging but cannot prove the 90% goal.
4. Make the baseline report expose timing diagnostics without changing shipping subtitle timing heuristics.
5. Use the current 0.8.1 algorithm as the baseline; only tune production Swift/C# after a repeated failure family is proven across samples and covered by red-green contract tests.

## Non-Goals

- Do not introduce a new ASR architecture in this first milestone.
- Do not bypass platform bot gates or login requirements without explicit user authorization.
- Do not commit media artifacts, cookies, credentials, or private subtitles.
- Do not tune for one video unless the failure signature repeats across language/content groups.

## Plan

1. Add first-onset diagnostics to Python scorecard with a failing test first.
2. Thread the metric into `run_scorecard_baseline.py` from `asr_words.json` and optional VAD speech segments.
3. Run the focused scorecard tests and then the full `tools/subtitle_timing_eval/tests` suite.
4. Generate or attempt the current scorecard baseline from existing ignored artifacts.
5. Use the resulting report to choose the next work item: sample evidence cleanup, source-decision scenario expansion, or production timing/selection tuning.

## Risks And Rollback

- Existing artifacts may be incomplete or absent; this should be reported as an evidence gap, not an algorithm failure.
- First-onset scoring must not double-count every cue onset. It is a sample-level diagnostic for the opening subtitle timing only.
- If baseline generation hits missing local dependencies, keep the code/test work verified and record the exact failing command.
- Rollback is simple but now has two layers: remove the scorecard metric changes and this plan file for eval-only rollback; revert the Swift/C# ASR continuation-token changes and their contract tests for production behavior rollback.
